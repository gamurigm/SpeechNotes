System Under Test

SpeechNotes — El SUT

Transcripción inteligente asistida por IA — graba, limpia, transcribe y formatea en tiempo real

🧠
Pipeline de 5 Modelos NVIDIA NIM
BNR → Parakeet ASR → Gemma (idioma) → Mistral (traducción) → Qwen 3.5 (formateo)
🌐
Frontend
Next.js 16, HeroUI, Socket.IO Client, Electron/Tauri
⚡
Backend
FastAPI, Socket.IO, pydub, FFmpeg, Logfire
🗄️
Datos
MongoDB, SQLite/Prisma, ChromaDB
🐳
Infra
Docker, Python 3.12, pnpm, GitHub Actions

Funcionalidades Principales

  • Grabación con VAD (Voice Activity Detection)
  • Transcripción en vivo via WebSocket
  • Upload de archivos de audio
  • Formateo IA con progreso en tiempo real
  • Búsqueda semántica RAG con ChromaDB
  • Chat contextual sobre notas
  • Procesamiento FFmpeg (normaliza, acelera)
  • App Desktop (Electron/Tauri)

Justificación: Sprint comprimido de 4 días. Se priorizan flujos críticos (grabación, transcripción) sobre cobertura total. Mejor pocos casos perfectamente documentados que muchos sin evidencia.

Entry Criteria

  • SUT compilable y ejecutable en entorno local
  • Backend respondiendo en /health → 200

  • Tasa de aprobación ≥ 90%

Herramientas

Stack de Herramientas

SonarCloud
Análisis estático continuo. Detecta vulnerabilidades, code smells y deuda técnica. Integrado con GitHub Actions para quality gates automáticos.
Análisis Estático

ESLint
Linter para el frontend Next.js/TypeScript. Configurado con eslint-config-next para core web vitals y reglas de TypeScript.
Linting

Pytest
Framework de testing Python para backend FastAPI. ~52 tests cubriendo 5 routers. Markers: smoke, regression, auth. Skip limpio si backend no accesible.
Backend Testing
🌲
Cypress
Tests E2E para el frontend. Simula interacción de usuario real: grabación, transcripción, navegación. Ejecuta en Electron headless.
E2E Testing
🃏
Jest
Tests unitarios para componentes React. Testing Library para renderizado y simulación de eventos. Cobertura de componentes críticos.
Unit Testing
📋
Jira Software
Gestión de incidencias y seguimiento de bugs. Flujo: To Do → In Progress → In Testing → Done. Épicas para SQA y métricas automáticas.
Gestión

CI/CD: GitHub Actions ejecuta SonarCloud en cada push a planning-SQA. El pipeline analiza backend/, src/, web/ y desktop/ con exclusiones para tests y archivos generados.

CP-03 — Iniciar y detener grabación

Crítica RF-03

Precondiciones:

  • Backend en puerto 9443, Frontend en 3006
  • Usuario autenticado, micrófono disponible

Pasos:

  1. Entrar a /dashboard
  2. Presionar botón de micrófono
  3. Aceptar permisos del navegador
  4. Hablar durante 10 segundos
  5. Presionar detener → Confirmar

Resultado Esperado:

  • Estado cambia a "grabando", contador avanza
  • Al detener: estado de procesamiento
  • Grabación se agrega a la lista

✓ PASO

Media RF-08

  • Usuario autenticado

Pasos:

  1. Entrar a /dashboard
  2. Abrir búsqueda global
  3. Buscar palabra existente

  5. Buscar palabra inexistente

Conclusiones

Conclusiones

Veredicto: Apto para Producción con Observaciones

El SUT se encuentra en un estado funcionalmente estable. El 92.86% de los casos de prueba fueron aprobados. Los 3 bugs Critical/Major fueron corregidos durante el sprint. Los 5 bugs restantes son de severidad menor y no bloquean el uso del sistema.

Hallazgos Clave

  • Funcionalidad Core Robusta: Auth, transcripción, upload, consulta, edición, búsqueda y chat pasaron todas las pruebas
  • 3 bugs UX Minor: No afectan funcionalidad pero impactan la calidad percibida
  • Mejora en Calidad de Código: SonarCloud permitió identificar y corregir code smells (−67%)
  • Base de Automatización: Pytest + Cypress proporcionan base sólida para regression testing futuro

Lecciones Aprendidas

📋
Planificación Reduce Ambigüedad
La matriz de rastreabilidad y diseño previo de casos redujo incertidumbre durante la ejecución.
🔄
Integración Temprana de SonarCloud
Detectar code smells antes de que se acumulen es más eficiente que refactorizar después.
🐛
Jira para Comunicación
El seguimiento de bugs en Jira facilitó la coordinación entre QA y desarrollo.
⏰
Sprints Acelerados
En sprints cortos, priorizar flujos críticos sobre cobertura total da mejores resultados.
