\section{Criterios de Aceptación y Rechazo}
\label{sec:criterios}

\subsection{Criterios de entrada (Entry Criteria)}
\label{subsec:entry}

\begin{itemize}
    \item Frontend operativo en \url{http://localhost:3006}.
    \item Backend operativo en \url{http://localhost:9443}.
    \item Base de datos SQLite disponible (\texttt{web/dev.db}).
    \item Usuario demo existente (\texttt{demo@speechnotes.app} / \texttt{demo123}).
    \item Microfono conectado (para pruebas de audio).
    \item Credenciales de API de NVIDIA NIM configuradas (para formateo IA y chat).
\end{itemize}

\subsection{Criterios de salida (Exit Criteria)}
\label{subsec:exit}

\begin{itemize}
    \item Los 14 casos de prueba han sido ejecutados.
    \item Todos los defectos de severidad \texttt{Critical} están cerrados.
    \item El informe de cierre ha sido generado con métricas consolidadas.
    \item El PDF del SQAP ha sido ensamblado y revisado.
\end{itemize}

\section{Matriz de Pruebas vs.\ Requisitos}
\label{sec:matriz-pruebas}

La matriz de rastreabilidad completa se presenta en el Apéndice~A. En resumen:

\begin{itemize}
    \item \textbf{10 requisitos funcionales} cubiertos al 100\%.
    \item \textbf{14 casos de prueba} distribuidos en 10 happy path, 3 error, 1 límite.
    \item \textbf{13 casos pasados} (92.86\%) y \textbf{1 caso fallado} (7.14\%).
\end{itemize}
